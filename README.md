## Notes

- The collection and environment files are kept under `postman/`.
- `reports/` is reserved for execution output and screenshots.
- `newman/` is reserved for HTML reports generated by Newman.

## Allure Reports

- Generate the report with `npm run test:api:allure` or `npm run test:api:devices:allure`.
- Open the main report with `npm run allure:open` after `npm run test:api:allure`.
- Open the device report with `npm run allure:open:devices` after `npm run test:api:devices:allure`.
- If you want to open it directly, use `allure open allure-report`.
- For the device-data run, use `allure open allure-report/devices`.
- If `allure` is not on your PATH, use `npx allure open allure-report`.

## Known Observations

- `dimensions.userLevel` and `dimensions.deviceType` can be returned as `null`.
- This is allowed by schema, but request/response consistency should be clarified.
- Endpoint is cached for 60 minutes, so content-related assertions should be designed carefully.
